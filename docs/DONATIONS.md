# DONATIONS.md — 支援・募金・トークンモデル設計書

原則：**透明性は寄付にも適用される。** 収入と支出の内訳は /donate ページで公開する。

## 1. 費用の三区分（混同を禁止する）

| 区分 | 内容 | 資金源 |
|---|---|---|
| A. 公共知識の提供 | 閲覧・探索・研究デスク・エクスポート | **永続無料**。運営費（月2千円級）は当面運営者負担＋寄付 |
| B. 高コストAI処理 | LLMによる反証生成・段階的読解拡張 | 利用者BYOキー（現行）→ 将来はコミュニティプール併設 |
| C. 人的専門作業 | 専門家レビュー・翻訳・教材化 | 将来の寄付・助成・機関連携。無償労働を前提にしない |

## 2. 受付手段（開設手順）

1. **GitHub Sponsors**（推奨・手数料0%）：github.com/sponsors で hand-shinya として申請 → Stripe連携 → 承認後 /donate ページのリンクが実働する
2. **Ko-fi / Open Collective**（代替）：Open Collective は会計が公開されるため区分Cの受け皿として適する
3. 開設までは /donate ページに「準備中」表示（現状）

## 3. コミュニティLLMトークンプール（設計のみ・実装凍結中）

目的：APIキーを買えない学生・利用者にもLevel 2を開く（公理5の完成形）。

設計要件（これを満たすまで実装しない）：
- 寄付で購入したAPIクレジットをサーバー側プールとして保持
- **濫用対策**：軽い認証（メールリンク等）＋日次上限（例：反証10回/日）＋用途限定（反証・段階的読解のみ。自由チャットは提供しない）
- プール残高と消費レートを /donate で公開
- 残高枯渇時は静かにLevel 0へ退化（サービス断ではない）

## 4. 将来の資金源候補

- 大学・図書館との機関連携（授業利用パッケージ）
- 公的助成（デジタル人文学・オープンサイエンス系）
- 教材・書籍化収入（CC-BY due クレジットの範囲で）

いずれの場合も区分Aの無料原則と AGPL-3.0 を交渉条件にしない（囲い込み提案は棄却する）。
