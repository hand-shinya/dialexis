★ v3.1 追加 — この岐路は特別だ 構えを「選ぶ」から「論考する」へ — 選択が射程・方向・結論を規定する

① 拡張可能なメニュー

AI提案の6構え系譜学・概念史・文脈主義／論証分析・概念分析・思考実験（足場）

固定メニューは足場であって檻ではない。閉じた選択肢集合は、それ自体が押し付けになる。

② 選択自体を論考する（AI特性の活用＝J層）

目的本当の問い「主体の位置」を問うのは、個の自立の可能性を擁護したいのか、それともその不可能性を暴きたいのか？ 目的が構えを変える。
背景ハビトゥス貴方の訓練は分析系？大陸系？ どの構えを「自然」と感じるかは、その傾きの現れ（Bourdieu）。
誤謬枠の前提「自立」を自明の善と前提していないか？ その前提が系譜学を最初から遠ざけているかも。
メタ認知自覚の促しこの構えを選ぶと、到達できる結論の範囲がここで決まる。それを承知で選びますか？
🤖 LLMのメタ認知（自己申告）：このメニューは私の学習データの偏り（大陸系・英語圏寄り）を帯びています。私は貴方を分析的伝統や非西洋の読解から逸らしているかもしれない。私が挙げていない構えを、貴方自身が疑ってください。
＝公理6（自己申告≠検証）・公理7（形式充足≠真理）・自己不信の可視化。オラクルでないことの最も深い形。

③ 哲学の優先順位で統べる

この対話は恣意でなく、哲学が重視する価値で導かれる：

一次資料への忠実妥当性 ≠ 健全性問いの真正性 前提の可視化反証可能性寛容の原理 統合不能の保持立場の再帰的自覚

論考が漂流しないための規準。どの構えも、これらに照らして射程と死角が測られる。

負荷とのバランス：論考はopt-in。素早く選びたい人には Guided Default が残り、深く構えたい人に論考モードが開く（Progressive Disclosure）。「重要な場面だと意識する」ことと「毎回長い対話を強制する」ことは違う。

決定サーフェスの7パターン Horvitz混合主導性・MS Human-AI指針・NN/gに接地

1. Fork Cards

一岐路＝カード群。各カードに固定4スロット〈射程・偏重・死角・結末〉。トレードオフを走査可能な構造に。comparison-table / pros-cons（NN/g）

2. Reach / Blind-spot メーター

各選択肢が捉える範囲と落とす範囲を被覆バーで。判定でなく被覆の提示。CDS証拠段階表示・仮説駆動支援（arXiv 2302.12389）

3. Differences Switch

共通地を畳み、射程と死角の分岐だけを前景化。comparison-table differences switch（uxpatterns.dev）

4. Progressive Fork Disclosure

主要な岐路を一行で。深部・サブ岐路は要求時に開示（選択過多を防ぐ・同時≤5）。NN/g Progressive Disclosure

5. Guided Default ＋ Escape Hatch

地図製作者の推奨を予め点す。常に上書き・中断可能。システムは岐路を自動で進めない。Horvitz sensible defaults / MS G9・G17

6. Consequence Preview（結末トレース）

岐路にホバーで「その先に至る問い・伝統」を確定前にプレビュー。可逆。Horvitz「誤推測のコスト最小化」

7. Option-Space Framing Header

各岐路に「これらが道。選ぶのは貴方。道具は地図を描き答えない」を明示。オラクル化を打ち消す。Lawfare「AIオラクルの神話」/ 一橋「地図製作者」

ツールUXの鉄則 採用/離脱の実データ（r/ObsidianMD・PKM・Elicit・Zotero）

✓装飾グラフを作らない。グローバルグラフは「見て楽しいが使えない毛玉／優先度・状態・鮮度を運ばない」。有界・局所の地図のみ。
✓構造を思考の途中で強制しない。分類は作業前に要求せず、創発させる。
✓既存ツールの「上に乗る」。Zotero/PhilPapers/Obsidianへ低摩擦で入出力。移行を強制しない。
✓可搬。消えても各ファイルは残る。素のMarkdown/BibTeX/JSONで持ち出せる（退場可能性）。
✓道具作りを仕事にしない。時間内滞在でなく「決定と成果」を最適化（M0 C7）。
✓判断を出す、データ投棄で終わらない。統合されたトレードオフを示し、不確実性は明示（Elicitの離脱理由の回避）。
