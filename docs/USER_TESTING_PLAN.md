# USER_TESTING_PLAN.md — 実在の哲学研究者による実ユーザーテスト設計

> **状態：PROPOSAL** — 2026-07-10。`PERSONA_EVALUATION.md` の構成的ペルソナを、実在ユーザーで検証する。
> 公理6（自己申告≠検証）・RESEARCH_REALITY §6（人間が判定を確定）を、設計の妥当性自体に適用する。

## 0. なぜ必要か
現状の設計・評価は公開資料への接地であり、**実利用者観察ではない**。ペルソナの共感/不満分布・
決定サーフェスの有効性・敷居の高さは、実在の研究者でしか検証できない。

## 1. 検証すべき仮説（反証可能な形で）
- H1（決定サーフェス）：Fork4の〈射程/偏重/死角/結末〉提示は、方法選択の質・自覚を高める。**反証**：利用者がカードを読まず既定を押す／選択理由を言語化できない。
- H2（ゲートの是正）：分析哲学者は「非歴史的」ゲートで自分を見出せる。**反証**：分析系利用者が依然「哲学＝大陸系」と感じ疎外される。
- H3（採用フロー）：explore→採用は手動再入力の負担を消し、探索とデスクの品質差を埋める。**反証**：採用したノードが使われず放置／採用先プロジェクトの鶏卵問題で離脱。
- H4（構造ビュー）：装飾グラフ撤去＋構造ビューは「研究を描く」に近づく。**反証**：利用者が構造ビューを読まない／旧グラフを懐かしむ。
- H5（LLMメタ認知）：AIの自己申告は信頼と自律を高める。**反証**：混乱・不信・無視される。
- H6（敷居）：研究デスクは学部生・在野者に複雑すぎる。**反証**：非研究者が難なく最初の成果に到達する。

## 2. 参加者（ペルソナ→実在リクルート・各2-3名）
- 博士課程院生（大陸系）／分析哲学ポスドク／日本思想・非西洋研究者／学部ゼミ生／在野の哲学愛好家／
  他分野の学際越境者。計12-18名。多様性クオータ：伝統3以上・非西洋1以上・非研究者1以上。

## 3. タスク（ペルソナ動線＝現実の作業）
1. 自分の研究テーマ/問いを1つ持参し、探索で定位する（論争地図・情報源）。
2. 気になった出典を採用してプロジェクトに入れる。
3. 読解の構えを選ぶ（ゲート→カード→決定）。選んだ理由を口頭で述べる。
4. 1つの論証をP1..Cで再構成する。
5. 成果を持ち出す（export／deepsearchプロンプト）。
※非研究者には1・2・5に限定した簡易版も用意。

## 4. 方法
- **モデレート付きThink-aloud**（画面共有・録画・同意取得）。各60-75分。
- 各タスク後に短い**共感/不満の口頭採取**＋SUS（システム使用性尺度）。
- 終了時に半構造化インタビュー：他ツール（PhilPapers/Zotero/ChatGPT等）との比較・改良提案。
- 観測指標：タスク成功率／初成果までの時間／離脱点／構造ビュー・カードの実閲覧（視線/操作ログ）／
  「地図でオラクルでない」が伝わるか。

## 5. 設計を反証しうる結果（先に宣言）
- 決定サーフェスのカードが読まれず既定押下が支配的 → Fork Cards の前提が崩れる。
- 採用ノードが放置される → 「接地して残す」価値が実作業に反映されていない（M0の有効性定義に不合格）。
- 非研究者が最初の成果に到達できない → 「小学生から」の使命は再設計を要する。

## 6. 倫理・運用
- 事前同意・録画同意・撤回可能・匿名化。BYO鍵は使わせない（テスト中はLevel 0のみ）。
- 謝礼。所要45-75分。オンライン（画面共有）中心、可能なら対面数名。
- 結果は本リポジトリに `docs/USER_TESTING_RESULTS_<date>.md` として記録し、改良は差分で測る（§dogfooding）。
