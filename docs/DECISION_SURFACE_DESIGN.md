## 0. 一行で

**Dialexis の UI の主組織原理を、システムの「位相の螺旋」から、ユーザーの「岐路（fork）の可視化」へ転換する。**
あらゆる研究の岐路で、選択肢を〈射程・偏重・死角・結末〉とともに示し、人間が根拠をもって選ぶ。
システムは地図を描くが、決して代わりに選ばない（proposer-of-options, human-ratified choice）。

## 1. なぜ再構想したか（トリガー）

1. **探索と研究デスクの品質差**（現物で確認）：`/explore` はコネクタ＋伝統ルーティングで自動的に価値を出す。
   一方 `/desk`・`/project` は 100% 手動 CRUD（type/confidence/origin をドロップダウンで全入力、辺も手動）＋
   cytoscape の cose 力学グラフ（色付き円）。**この可視化はデータモデルを描いており、研究を描いていない**。
   → M0 C1/C2 違反（可視化の目的化）・§2 違反（手作業負担）。
2. **二つの独立体系**：半田様は事実上、二つの哲学研究 AI 体系を育ててきた。
   - **一橋体系**（`Main/論考/AI拡張弁証法/hitoysubashi/`）＝認識論的エンジン：E1-E2-E3 計算論的弁証法、
     T1-T5 典拠階層、AutoDiversify、Σ1-Σ6 出力契約、量質転化◎/創造的緊張★、博士方法論の全ツールキット
     （プロブレマティーク／問題化／考古学・系譜学／概念史／ケンブリッジ学派／ブルデュー場理論／
     徴候的読解／脱構築／概念創造）。ベンチマーク＝福山圭介『アルチュセール思想の理論的構造と社会的生成』
     （一橋大博士論文・内在的理論構造＋社会的生成の二部弁証法）。
   - **Dialexis** ＝接地されたインフラ：キーレス・退場可能・provenance・研究過程グラフ。
   一橋＝ENGINE(推論)だが ¬PERSIST ∧ ¬GROUND ∧ ¬EXIT。Dialexis＝SUBSTRATE(接地・永続・退場可)だが ¬ENGINE。
   **相補構造**。

## 2. 中核の止揚：Dialexis ＝ 接地された・再開可能な・弁証法プロセス基層

Dialexis を「コネクタ付き研究デスク」ではなく、**一橋の弁証法方法論がその上で走る、唯一の《永続的・接地的・
決定論的な基層》**として構想し直す。弁証法の産物すべてを、出典に接地されたグラフ資産として残す。

- **保存**：Dialexis の退場可能性・キーレスコネクタ・provenance/retrieved_at・人間が裁定・研究過程グラフ・
  伝統ルーティング／一橋の E1-E2-E3・T1-T5 Evidence Ledger・AutoDiversify・Σ6 自己批判・方法論ツールキット。
- **否定**：Dialexis の受動性（思考しない書庫）／一橋の消散性（接地なき自己申告の出力・provenance DB なし・退場不能）。
- **高次化**：弁証法を「消える出力」から「残る・接地された・再開可能なプロセス資産」へ。

### Dialexis が存在する唯一の理由（他の LLM プロダクトに構造的に不可能なこと）
誰でも何でも AI に聞ける時代に希少なのは、**答えではなく「接地され・構造化され・出典を帯びた素材」と
「確定/仮説の判別」**。Dialexis は各主張を **外部から検証可能**にできる（主張→provenance 行→ライブコネクタ→
retrieved_at）。＝ **弁証法的哲学研究のための外部検証基層＝接地コンパイラ**。孤独な博士課程の弁証法を、
止めて再開でき・出典に接地され・累積し・共有できる知的コモンズに変える。

### 橋渡し資産
一橋同梱の `philosophy_knowledge_v0.1.sqlite` は、一橋方法論が既に Dialexis の語法で結晶したデータモデル：
`evidence`(T1-T5)／`textunits`(標準ロケータ＋tradition/method/region/period)／`claims`(**fals_cond=反証条件**)／
`counterarguments`([i]/[e] 軸)／`comparative_schema`(思想家マトリクス)／`terms`(語彙系譜)／
`bibliography_bibtex/csl`(Zotero)。WORKFLOW_COVERAGE で ❌ だった B/D/G2/I4 がここに設計済。
**加法的に吸収**すべき（再発明しない）。

## 3. UI の主組織原理：決定サーフェス（map not oracle）

半田様の警告：「システム側の勝手な UI や考え方で組むと失敗する」。自己批判として、当初案の「6位相の螺旋」は
一橋の Σ1-Σ6（AI の**出力**構造）由来であり、ユーザーの**決定**構造ではなかった（RESEARCH_REALITY §1：研究は
「網羅的でなく論証駆動で目的的」＝非線形）。→ **螺旋は背景の痕跡に降格。前景はユーザーの「岐路」。**

決定サーフェスの原理：あらゆる岐路で、選択肢を〈**射程**（捉えるもの）・**偏重**（前景化するもの）・
**死角**（外れるもの）・**根拠**（選ぶ理由）・**結末**（行き着く先）〉とともに可視化し、人間が選ぶ。
選択は**根拠つき決定ノード**（Decision Ledger）として残る。これは一橋「AI を神託者でなく地図製作者に」＋
RESEARCH_REALITY §6「証拠の提示者・人間が判定を確定」の UI 化。

## 4. 改訂された研究の岐路（外部の実在ワークフローで検証済）

3方向の外部 web 調査（実在の哲学ワークフロー／研究ツール UX／決定支援 UI）で岐路モデルを検証。
判定：**確認3・修正2・反証1・追加2**。

| 岐路 | 判定 | 根拠（出典） |
|---|---|---|
| **1 定位入口**（SEP/CiNii/PhilPapers/指導教員） | 強く確認 | Maynes がこのメニュー。PhilPapers は browse↔search 二モード（Illinois State LibGuide） |
| **2 着目点**（論争地図のどこ） | 確認 | 二次文献で「立場と論点」を見つけ位置を定める（Maynes） |
| **3 読解の構え（方法/レンズ）** | **修正（最大）** | 分析哲学者は方法を自己同定せず issue を非歴史的に扱う。決め手は先行する一問＝**「概念を非歴史的実体と見るか、歴史的産物と見るか」**（newAPPS）。→ 7方法の平置きメニューは条件化（ゲート）せよ |
| **4 貢献の型**（解釈/批判・創造/統合） | 確認 | 批判＋創造は融合（Daily Nous / Pryor は論証されたテーゼを要求） |
| **0 問いの成立** | 修正 | 「プロブレマティーク」の名札でなく**「面白い ⇄ 論文になる/扱える」の緊張**として経験（C.Thi Nguyen, Daily Nous）＋扱いやすさゲート（Grad Coach）。→ 扱いやすさメーターに |
| **統合 vs 保持** | 弱い→降格 | 独立岐路として実証弱。Fork 5 内の一選択へ |
| **出力/Handoff** | 反証→再分類 | 出典に射程トレードオフを持つ研究の岐路でなく、**執筆/持出しの工程**（Pryor 明晰性規約） |

### 追加された岐路（実データが要求）
- **反論ドック（objection/steelman）＝最大の見落とし**：「決定的反論の見落とし」不安は末尾チェックでなく
  **セッション全体を貫く継続的活動**（Pryor／Suber の objection-reply 課題／steelmanning）。**常時表示の独立サーフェス**。
- **語彙精緻化ループ**：「検索は与えた語彙の質までしか良くならない…関連が無ければ新しい語を試せ」（Maynes）。
  反復ループ・**捨てた語も記憶**。

### 逐次岐路と常時サーフェスの区別
- 逐次：1問いの成立 → 2定位入口 → 3着目点 → 4読解の構え → 5貢献の型
- 常時（セッション全体）：◆反論ドック・◆語彙ループ
- 工程：↧出力 Handoff（用途×粒度×形式）／背景：螺旋の痕跡（決定履歴）

## 5. Fork 4「読解の構え」の格上げ ― 構成的契機（半田様の深化・最重要）

構えの選択＝**プロブレマティークの選択＝射程の決定＝到達可能な結論の先取り**。研究全体で最高レバレッジの契機。
ポータルは「ここは重要な場面だ」と意識し、メニューを差し出すだけで済ませない。三層：

### ① 拡張可能なメニュー（AI 提案 ＋ ユーザー独自）
AI の固定メニューは**足場であって檻ではない**。ユーザーは自分の構えを定義・合成できる（例：系譜学×精神分析の
三角測量、独自の読解枠）。閉じた選択肢集合はそれ自体が押し付けになる。

### ② 選択自体を論考する（AI 特性の活用＝J層の実装）
WORKFLOW_COVERAGE の **J層（反省層：真の問い・語の背景・利用者バイアス）を Fork 4 に実装**。会話で汲むもの：
- **目的**：表層クエリの奥の「本当の問い」
- **背景**：Bourdieu のハビトゥス＝何に傾きがちか
- **誤謬**：枠組みに潜む誤り・無自覚な前提
- **メタ認知**：「この選択が結論を縛る」ことの自覚の促し
- **LLM のメタ認知（最重要）**：AI が自分の枠の押し付けを**自己申告**する。
  例：「このメニューは私の学習データの偏り（大陸系・英語圏寄り）を帯び、貴方を分析的伝統や非西洋の読解から
  逸らしうる。私が挙げていない構えを疑ってください」。＝**公理6（自己申告≠検証）・公理7（形式充足≠真理）・
  自己不信の可視化**。オラクルでないことの最も深い形（AI が自らの誘導を反省して開示する）。

### ③ 哲学の優先順位で統べる
論考は恣意でなく、哲学が重視する価値で導かれる：一次資料への忠実（T1-T5）／論証の妥当性 ≠ 健全性／
問いの真正性／前提の可視化／反証可能性／寛容の原理／統合不能の保持／立場の再帰的自覚（Bourdieu reflexivity）。
どの構えも、これらに照らして射程と死角が測られる。

### 負荷とのバランス
論考は **opt-in**（Progressive Disclosure）。素早く選びたい人には Guided Default が残り、深く構えたい人に
論考モードが開く。「重要な場面だと意識する」ことと「毎回長い対話を強制する」ことは違う（M0 §2）。

## 6. 決定サーフェスの7 UI パターン（外部指針に接地）

1. **Fork Cards** — 一岐路＝カード群、各カードに固定4スロット〈射程・偏重・死角・結末〉（NN/g Comparison Tables / pros-cons）
2. **Reach/Blind-spot メーター** — 捉える/落とす範囲を被覆バーで（**判定でなく**）。CDS 証拠段階表示・仮説駆動支援（arXiv 2302.12389）
3. **Differences Switch** — 共通を畳み差分だけ前景化（uxpatterns.dev comparison-table）
4. **Progressive Fork Disclosure** — 主要な岐路を一行で、深部は要求時（同時≤5・選択過多防止）（NN/g Progressive Disclosure）
5. **Guided Default ＋ Escape Hatch** — 地図製作者の推奨を予め点す。常に上書き・中断可能。自動で進めない（Horvitz / MS G9・G17）
6. **Consequence Preview（結末トレース）** — 岐路にホバーで「その先に至る問い・伝統」を確定前にプレビュー。可逆（Horvitz「誤推測コスト最小化」）
7. **Option-Space Framing Header** — 各岐路に「これらが道。選ぶのは貴方。道具は地図を描き答えない」を明示（Lawfare「AI オラクルの神話」／一橋「地図製作者」）

出典：Horvitz, *Principles of Mixed-Initiative User Interfaces*（CHI 1999, erichorvitz.com/uiact.htm）／
Microsoft *Guidelines for Human-AI Interaction*（18項）／NN/g（Progressive Disclosure, Comparison Tables）。

## 7. ツール UX の鉄則（採用/離脱の実データ）

- **装飾グラフを作らない**：グローバルグラフは「見て楽しいが使えない毛玉／優先度・状態・鮮度を運ばない」
  （r/ObsidianMD 総意・codeculture）。有界・局所の地図のみ。
- **構造を思考の途中で強制しない**：分類を作業前に要求せず、創発させる（PKM paradox・Roam→Tana）。
- **既存ツールの「上に乗る」**：Zotero/PhilPapers/Obsidian へ低摩擦で入出力。移行を強制しない。捕捉10秒超で続かない。
- **可搬。消えても各ファイルは残る**：素の Markdown/BibTeX/JSON（退場可能性＝GENESIS 公理7）。
- **道具作りを仕事にしない**：時間内滞在でなく「決定と成果」を最適化（M0 C7）。
- **判断を出す、データ投棄で終わらない**：統合されたトレードオフ＋不確実性の明示（Elicit の離脱理由の回避）。

## 8. 出力・持ち出し層（Handoff）― 出力＝入力

出力は終端でなく次の文脈への橋。各位相で取り出せ、〈**用途 × 粒度 × ファイル形式**〉で形が決まる。
provenance・retrieved_at・confidence が同梱され、受け手（他 AI／DB）は「確定/仮説の判別と出典」を継承し、
再ハルシネーションせず「続きから」考える。
- 用途プリセット：別 AI 継続プロンプト（.txt）／論文の一節（.md/.docx）／Obsidian ノート（YAML＋[[wikilink]]＝
  KB 流入）／DB 投入（.jsonl/.csv/.sqlite）／Zotero 文献（.bib/.json）／ゼミ配布（.md/PDF）。
- 粒度：全体／1サイクル／1論証／主張＋その出典／パノラマのみ／文献のみ。

## 9. 保持する公理（GENESIS/M0 と整合）

- **退場可能**：ビルド工程なし・BYO 鍵・単一 SQLite・素の可搬ファイル。
- **自己不信**：AI は証拠たりえず人間が裁定（一橋 knowledge_is_non_evidence ＝ GENESIS 公理6）。
- **手間ゼロ**：採用で自動接地・手入力を排す（M0 §2）。
- **接地**：各主張に出典＋retrieved_at＋位相。
- **可視性**：沈黙する失敗を作らない（M0 A1）。
