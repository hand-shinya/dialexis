# USER_GUIDE.ja.md — 使用手順書（日本語）

対象読者：Dialexisを使うすべての人。哲学の知識は前提としません。[English version](USER_GUIDE.md)

## Dialexisは何をしてくれるのか

「自由って結局なんだろう」「AIに責任はあるのか」——そんな問いが浮かんだとき、Dialexisは**答えを与えるのではなく、あなたの問いを研究に変える**手伝いをします。世界中の学術情報源への窓口と、考えを整理する研究デスクが、無料・登録不要・APIキー不要で使えます。

## 1. 探索（トップページ）

検索窓に哲学者・著作・概念を入れます（例：`Karl Marx`、`自由`、`Sein und Zeit`）。

- 人物なら：生没年・主要著作・影響関係（Wikidata）、要約（Wikipedia）、原典全文へのリンク（Wikisource / Project Gutenberg）、関連論文（OpenAlex / Crossref）が一画面に集まります。
- すべての情報に「**どこから・いつ取得したか**」のバッジが付きます。バッジのない情報はこのシステムには存在しません。
- 赤いバッジは「その情報源がいま応答していない」ことを隠さず示しています。故障ではなく設計です。

## 2. 研究デスク

「新しい研究を開始」にタイトルと**最初の問い**を入れます。問いは自動的に最初のノードになります。

以下の種類のノードを追加し、矢印でつないでいきます：

| ノード | 使いどき |
|---|---|
| 問い | 調べたいこと。研究の出発点 |
| 主張 | 「〜である」と言いたいこと |
| 根拠 | 主張を支える資料・事実（出典URLと取得日を必ず添える） |
| 反証 | 主張に対する反対意見・反例 |
| 未確認事項 | まだ確かめていないこと。**ここに正直であることが研究の質** |
| 解釈 | 資料の読み方についての自分の見解 |
| 判断 | 採用／保留／棄却の決定 |

各ノードには**確度**を付けます：確定／高蓋然／未確認／解釈仮説／思弁。「まだ思弁」と認めることは恥ではなく、このシステムの美徳です。

**書き出し**：研究はいつでも Markdown（人間向け）/ JSON-LD（機械向け）で完全に持ち出せます。レポート・論文の下書き、指導教員への提出、他ツールへの移行に使えます。

## 3. 反証エンジン

主張を入力すると、6つの専門的視点（文献学／翻訳・概念史／分析哲学／思想史／科学技術／社会・制度）からの「尋問リスト」と、対立文献の候補が返ってきます。

- **APIキーなし（Level 0）**：チェックリスト＋文献検索。これだけでも自分の主張の穴は相当見つかります。
- **APIキーあり（Level 2）**：各視点からの具体的な反証文が生成されます。AI生成物には必ず「未確認」の標識が付きます。

## 4. 新着監視（Watcher）

「Karl Marx」を登録しておくと、新しい論文・著作・資料が出るたびに検出されます。**マルクスのような古典的著者ですら、新資料の発見は起こりうる**——それがこの機能の思想です。登録後は自動で定期照会されます（「今すぐ照会」で手動確認も可）。

## 5. 段階的読解

同じ概念を7段階の深さ（小学生→専門家）で読めます。まずは「自由」を小学生レベルと大学院生レベルで読み比べてみてください。哲学が暗記科目ではないことが分かります。APIキーを設定すると任意の概念に拡張されます。

## 6. 設定（鍵スイッチ盤）

Anthropic / OpenAI / Gemini / Ollama（ローカル・無料）のAPIキーを設定すると、AI機能が昇格します。

- 鍵は**このブラウザの中にだけ**保存されます。サーバーには一切送信・保存されません（AI呼び出し時にのみ中継されます）。
- 同ページの「AI Transparency Ledger」で、このサーバー上でAIがいつ・何をしたかを誰でも確認できます。

## 7. こんな使い方（ワークフロー例）

**中学生**：探索で「自由」→ 段階的読解で自分のレベルの説明を読む → 「本当に自分で選んでいるのか？」と思ったら研究デスクに問いとして保存 → 反証エンジンで自分の考えを試す。

**大学院生（例：マルクスの「非有機的肉体」概念を研究）**：
1. 探索で Marx → Wikisource原典・OpenAlex関連論文に到達
2. 研究デスクに問い「LLMは知的な非有機的器官と言えるか」を立てる
3. 原典箇所を根拠ノードに（出典・取得日つき）、訳語の危険（Leib/Körper）を未確認事項に
4. 反証エンジンで6視点の反証を取得、反証ノードとして採録
5. Watcher に「inorganic body Marx」を登録して新研究を自動監視
6. Markdownで書き出して指導教員に提出
