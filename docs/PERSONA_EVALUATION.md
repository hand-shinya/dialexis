## 他システム・従来手法との差異（と、あるべき差異）
- Google/汎用検索：論争地図・接地・確度 vs リンクの束 →「探す」でなく「論争に定位する」。
- ChatGPT/Perplexity：外部検証可能な接地・オラクルでない決定サーフェス vs 自信ありげな単一回答 → 答えでなく選択肢の地図を接地して示し人間の判断を残す。他AIを指揮する接地層。
- PhilPapers/SEP：それらを入口に使い研究過程を残す vs 索引そのもの → 索引でなくプロセス基層（未接続が最大の穴）。
- Zotero：論証・確度・研究過程 vs 書誌のみ → 置換でなく上に乗る（export.bib必須）。
- Obsidian/Roam：装飾グラフの失敗を避け接地された論証構造（現状まだcoseグラフ残存）→ 局所/構造ビュー、可搬出力。
- Elicit：解釈的問いに接地・判断を出す vs データ投棄 → 判断を出す。
- 一橋GPT：接地・永続・再開・外部検証 vs 消える自己申告出力 → 一橋の方法を残る形にする。
- 従来の手作業（SEP→PhilPapers→ノート→Zotero）：動線を一つの接地グラフに畳み方法選択を可視化し他AIへ手渡す → 研究過程を検証可能・再開可能・共有可能に。

**あるべき差異の一点**：「答える／索引する／管理する」のいずれでもなく、**研究過程を接地して残し、選択の地図を示し、人間の判断を残す接地コンパイラ**。

## 横断的総合（優先度つき）
**共通の共感**：接地／確度分類／退場可能／地図でオラクルでない／LLMメタ認知。

**共通の不満（重い順）と推奨改良**：
1. PhilPapers未接続（§1主力）— 早く着手（API申請に時間）。
2. 決定サーフェスと旧・手動デスク/装飾グラフの二重 — **決定サーフェスへ統一**（最優先・スキーマ不要・一貫性回復）。
3. 論証再構成が全手動・deepsearchと非連結。
4. 反論ドック未実装。
5. オンボーディング/「問いから始める」入口の欠如。
6. Zotero出力なし。
7. モバイル・非西洋の論争地図/参照単位。

**着手順（承認済 2026-07-10）**：① 決定サーフェスへの統一＋装飾グラフ撤去 → ② export.bib → ③ PhilPapers → ④ 反論ドック → ⑤ 「問いから始める」オンボーディング。
